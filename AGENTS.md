# root-rule
## Commit Message Instructions
git diffの内容に基づき、以下のルールを厳守してコミットメッセージを生成してください。

1. **仕様**: [Conventional Commits](https://www.conventionalcommits.org/ja/v1.0.0/) に従うこと。
2. **フォーマット**: `<type>: <description>` の形式とし、**完全に1行のみ**で出力すること（詳細行は不要）。
3. **言語**: `<description>` は日本語で簡潔に書くこと。

### Typeの基準
- feat: 新機能
- fix: バグ修正
- docs: ドキュメントのみ
- style: フォーマット（コードの動作に影響しない）
- refactor: リファクタリング
- perf: パフォーマンス改善
- test: テストの追加・修正
- chore: ビルドやツール、ライブラリの変更
