# 回答言語およびプロジェクトドキュメント保存ルール

## 基本方針
- 特に指示がない限り、日本語で回答する。
- 以下のドキュメントも**日本語で作成する**：
  - 実装計画 (Implementation Plan)
  - 修正内容の確認 (Walkthrough)
  - タスクリスト (Task List)

## ドキュメント生成ルール
- **複雑なコード変更や新規機能の実装を伴わない** 単純な質問や情報取得タスクの場合、**ドキュメント (`task.md`, `implementation_plan.md`, `walkthrough.md`) は生成しない**こと。
- 特に、モデルが **Fast** の場合は、ドキュメント生成の省略を推奨する。
- ドキュメントを生成する必要がある場合は、ユーザーの明示的な許可を得ること。

## ファイル構成
- チャット内で作成した以下の3つのファイルを保存する：
  - `task.md`
  - `implementation_plan.md`
  - `walkthrough.md`

## 保存場所
- プロジェクト内の `docs` フォルダの下に新しいフォルダを作成する。
- フォルダ名はこのチャットのトピックを表す適切な英語名とし、スペースはアンダースコアで区切る。

## Gemini Added Memories
- ユーザーは、パブリックフォルダの使用と Next.js へのアセットのインポートの比較を求めており、フロントエンドのアセット管理に関するアーキテクチャガイダンスの必要性を示唆しています。
- ユーザーは、管理画面で自分のカメラではなく別のデバイス（スマートフォン）からのカメラフィードを表示できるかどうかを知りたいと思っています。また、2 台の iPhone を所有していることから、セキュリティ上のハードルについても質問しています。
- ユーザーは、最初は外部 WebRTC サービスを使用せずに、同じローカルネットワーク（Wi-Fi）上のデバイス間でカメラフィードを表示してみたいと考えています。
- ユーザーは、Next.js アプリケーションを Firebase Hosting にデプロイし、カメラストリーム アーキテクチャに Cloud Storage と Firestore を使用する予定です。
